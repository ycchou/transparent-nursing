# 護理職場透明化運動

> **讓真實的職場資訊，被下一個護理師看見。**

🌐 **網站**：https://ycchou.github.io/transparent-nursing/

---

## 這是什麼？

一個讓護理師**匿名分享自己職場真實樣貌**的平台 — 你的薪水、護病比、值班制度、加班費政策、工作氣氛，全部開放給其他護理師參考。

同時，我們把散落各處的**官方公開資料**（三班護病比、醫事人力、醫院財務、勞檢/性平/職安違規、醫院評鑑）整理成可查閱、可比較、可深入單一機構的資料庫。

一句話：**把護理職場的「事實」——不論來自同行分享或政府公開資料——攤在陽光下。**

## 為什麼要做這個？

求職時，你可能聽過：

- 「那間醫院薪水很高啦，但你不知道夜班費被切了一半」
- 「A 醫院 ICU 1:2 護病比，B 醫院寫得很漂亮其實是 1:4」
- 「進去之後才發現加班從來不報，反正主管會 hold 住」

**這些資訊明明對護理師至關重要，卻完全沒有公開比較的管道。** 醫院官方介紹只放福利、薪資範圍給的是上限、PTT 評論又無法量化。而政府其實公開了很多硬資料（護病比、人力、財務、違規），只是分散、難讀、沒有人把它們對到「同一家醫院」上。

於是這個平台把兩者接起來：**社群的真實體感 × 官方的硬數據**，讓你查一家醫院時能一次看全。

## 平台功能

網站分為兩大類：**社群分享**（護理師投稿）與**官方資料**（政府公開資料整理）。

### 📊 [分享平台](https://ycchou.github.io/transparent-nursing/platform.html)
600+ 筆來自全台醫院的真實職場資料。10 大護理工作場域分眾呈現：

> 病房 / 加護病房 / 急診 / 手術房 / 門診 / 診所 / 洗腎室 / 精神科 / 檢查介入 / 其他

可依**地點、機構等級、工時、推薦指數、加班費合規**多重篩選，看到每一筆的：護病比、薪資、年資、推薦指數、個人短評等。

### 🏥 [機構總覽](https://ycchou.github.io/transparent-nursing/hospital.html)（旗艦整合頁）
輸入一家醫院，**一頁看全**該機構的所有面向，以「機構代號」為錨串接五大資料源：

- **三班護病比**（VPN 登錄）
- **醫院財務**（健保署）
- **人力監控**（衛福部醫事人力監測）
- **分享平台**眾包職場資料
- **違規紀錄**（勞檢／性平／職安）

涵蓋 482 家評鑑合格醫院。**多院區醫院**（如臺北市立聯合醫院、耕莘醫院）以**院區頁簽**分別呈現護病比與人力監控，切換院區不必一路下拉。

### 🩺 [三班護病比](https://ycchou.github.io/transparent-nursing/nurse-ratio.html)
健保特約醫院於 VPN 登錄的**白班／小夜／大夜**急性一般病床護病比，476 家、民國 112/07–115/05 逐月資料。依醫院層級套用達標標準，分級標示合規情形；多院區各院區分別呈現。

### 💰 [醫院財務](https://ycchou.github.io/transparent-nursing/financials.html)
健保署「醫院財務資訊公開」整理，255 家醫院的整體損益、醫務利益率、醫務收入等，可**全院排序比較**，點入單一醫院看**多年趨勢圖**。

### 👥 [人力監控](https://ycchou.github.io/transparent-nursing/personnel.html)
衛福部「醫院醫事人力持續性監測結果」整理，472 筆（含院區）、民國 108/07–115/05 逐月。呈現 **13 種醫事職類**（醫師、護產、藥事、放射、檢驗、呼吸治療、物理／職能／語言治療…）的實際人數與各類病床數變化：

- 單院時間序列 + 全國逐月儀錶板
- 護產等基數大的職類，Y 軸自動縮放讓月變化清楚可辨
- 未填報之月份線段中斷、**不補值**，忠實呈現
- 多院區以院區頁簽切換（比照護病比）

### ⚠️ [違規紀錄](https://ycchou.github.io/transparent-nursing/records.html)
整合三支政府公開的醫療機構違規資料，並比對到對應醫院：

- **勞檢**（勞動基準法）
- **性平**（性別平等工作法）
- **職安**（職業安全衛生法）

哪間醫院被罰過幾次、違反哪些條款、罰鍰多少，一目了然。

### 📈 [統計摘要](https://ycchou.github.io/transparent-nursing/stats.html)
社群分享資料的全景視覺化：投稿分布、機構類別 × 工作場域交叉、每週工時分布（55hr+ 紅色警示）、加班費合規、推薦指數、年薪分布（Q1／中位數／Q3）、地點分布。

**還整合三份官方報告**作為對照基準：
- 衛福部 112 年醫院護理服務量調查（薪資、夜班費、離職率等）
- 護理全聯會 111 年薪資調查（依縣市、層級、年資）
- 護理全聯會 114 年人力監測指標（職場流向、年資、淨增減）

### 🧮 薪資百分位試算
在分享平台輸入你的年薪 → 立刻告訴你**在同條件護理師中排第幾百分位**。可隱藏具體數字後**產生分享圖到社群**，跟同行討論你的薪水合不合理。

### 📝 [填寫表單](https://ycchou.github.io/transparent-nursing/participate.html)
10 個工作場域對應 10 份表單，目前**加護病房**、**洗腎室**、**門診**、**診所**、**其他**已上線，其餘陸續開放中。

## 匿名性與隱私 — 重要承諾

這個平台的設計理念是「**完全匿名、不可追溯、送出即公開**」：

| 我們不收集 | 為什麼 |
|---|---|
| **IP 位址** | 不留下任何網路足跡 |
| **Email** | 不要求帳號登入 |
| **Cookie / 登入狀態** | 不綁定任何身分 |
| **設備指紋** | 不用瀏覽器特徵反推誰是誰 |
| **時間戳記精確到秒** | 防止與其他資料比對 |

**送出後資料立即公開**，沒有後台審核延遲。但同時也代表：

⚠️ **送出後無法撤回任何單一筆資料** — 因為我們不知道哪一筆是你的。請在送出前審慎評估你填的內容。

## 怎麼參與？

### 我想分享我的職場
直接到 [填寫表單](https://ycchou.github.io/transparent-nursing/participate.html)。3-5 分鐘填完即可。完全匿名。

**建議**：

- 你的單位若全院只有 1-2 人填寫，被同事/主管推測出來的機會就高。可選擇模糊填寫機構名稱（例如「某醫學中心（北部）」）
- 短評欄位避免寫到具體事件、人名、特定時間點
- 薪資數字過於精準（例如 928,450 元）會比區間值更易被認出

### 我想幫忙改善這個平台
- 在 [GitHub Issues](https://github.com/ycchou/transparent-nursing/issues) 回報 bug 或提建議
- 直接送 Pull Request
- 把這個網站介紹給身邊的護理師

## 技術細節（給開發者）

完全靜態網站，無後端、無 build step：

- 純 HTML / CSS / vanilla JS（ES module，以 `?v=` query 破快取）
- [Chart.js](https://www.chartjs.org/) 視覺化 / [PapaParse](https://www.papaparse.com/) CSV 解析 / [html2canvas](https://html2canvas.hertzen.com/) 分享圖生成 — 全部 CDN 載入
- 託管：GitHub Pages（PWA，可加入主畫面）
- 社群表單：目前用 Google Forms，未來規劃自建（部分已上線）

**頁面結構**：多頁（每個資料源一頁）+ `hospital.html` 機構總覽整合頁。共用檢視邏輯抽成 `*-view.js`（如 `nurse-ratio-view.js`、`personnel-view.js`、`financials-view.js`）供各頁與機構總覽共用，單一來源。

**資料前處理**（`tools/`，離線執行，產出 `data/*.json`）：

| 工具 | 作用 |
|---|---|
| `build-nurse-ratio.py` | VPN 三班護病比 ODS → `nurse-ratio.json`、`hospitals-merged.json`（含多院區拆分） |
| `build-personnel.py` | 醫事人力監測 PDF → `personnel/{id}.json`、`personnel-index.json`、`personnel-aggregate.json`（僅拆護病比認定的真多院區） |
| `fetch-hospital-financials.py` | 健保署財報 → `hospital-financials.json` |
| `build-violations-map.py` | 違規機構名稱 → 機構代號對照表 |
| `extract-hospitals.py` | 評鑑合格名單 PDF → 醫院清單 |

多院區處理：同一機構代號涵蓋多院區時（北市聯醫、耕莘），以護病比 `nurse-ratio.json` 的院區身分為權威，護病比與人力監控皆分院區呈現、彼此一致。

**本地開發**：
```bash
python -m http.server 8000  # 或 npx serve .
```
開啟 http://localhost:8000
